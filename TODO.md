Вот несколько **интересных и полезных** идей, которые можно добавить в твой `aicli`, чтобы проект стал значительно круче и современнее (2025–2026 уровень).

Я разбил их по категориям: от простых (можно сделать за 1–2 часа) до продвинутых (более амбициозные).

### 1. Быстрые и полезные улучшения (рекомендую начать с них)

- **`/status`** (уже сделали) → расширить:
  - Показывать температуру, top_p, max_tokens из конфига
  - Проверять скорость ответа (latency)
  - Показывать версию сервера (если LocalAI/Ollama отдаёт)

- **Поддержка нескольких провайдеров** (LocalAI + Ollama + OpenAI):
  - В конфиге добавить `provider: localai | ollama | openai`
  - Автоматически менять эндпоинты и формат запросов

- **Markdown-рендеринг ответов** в терминале (очень красиво)
  - Использовать библиотеку `github.com/charmbracelet/glamour` или `github.com/rivo/tview`
  - Код, списки, таблицы, заголовки будут красиво подсвечиваться

- **Многострочный ввод** (удобно для больших промптов)
  - По Ctrl+D или `/end` завершать ввод
  - Или режим `vi`/`emacs` в readline

- **`/branch <name>`** или ветвление чатов
  - Создавать ветки разговора (как в Git), чтобы экспериментировать без потери истории

### 2. Средний уровень (очень повышают ценность)

- **RAG над своими файлами / кодовой базой**
  - Команда `/index <dir>` — индексировать папку (используя embeddings от LocalAI или простое TF-IDF)
  - `/ask <вопрос>` — искать релевантные файлы и добавлять в контекст
  - "Chat with your codebase" — один из самых популярных сценариев сейчас

- **Генерация и выполнение shell-команд**
  - Режим `/shell` или специальный промпт: "напиши команду для …"
  - Показывать команду → спрашивать подтверждение перед `exec.Command`
  - Очень мощно и опасно одновременно (делай с осторожностью)

- **Сохранение чатов в разных форматах**
  - `/export <name> --format md|json|txt`
  - `/import` из Markdown

- **Автоматическое именование чатов**
  - При `/new` или после первого сообщения просить модель придумать хорошее название чата

- **`/image` или мультимодальность** (если LocalAI поддерживает)
  - Генерация изображений + просмотр в терминале (kitty protocol или просто ссылка)

### 3. Продвинутые / "вау" фичи (агентный уровень)

- **Инструменты (Tools / Function Calling)**
  - Позволить модели вызывать функции: поиск в файлах, чтение URL, запуск команд, калькулятор и т.д.
  - Это уже почти настоящий агент

- **Режим "Code Agent"**
  - `/code review <file>`
  - `/refactor <функция>`
  - `/write test` — генерировать тесты
  - Автоматически редактировать файлы в проекте (с подтверждением)

- **Поиск по истории всех чатов** (`/search "что я спрашивал про docker"`)

- **Voice input** (голосовой ввод) через `sox` или `whisper.cpp` локально

- **Мульти-сессия / несколько окон**
  - Поддержка нескольких чатов одновременно (вкладки через tview или просто переключение)

- **Интеграция с Git**
  - Автоматически добавлять diff текущей ветки в контекст
  - `/commit "описание"` — генерировать сообщение коммита

### 4. Качество жизни (QoL)

- Цветные темы (light/dark)
- Настраиваемые горячие клавиши
- Прогресс-бар для длинных ответов
- Автосохранение черновика сообщения
- Статистика использования (сколько токенов потрачено, если сервер отдаёт)
- Поддержка `.env` + флаги командной строки (`--model`, `--host`, `--chat`)

3. **RAG над папкой** (даже простой вариант) — это то, за что люди сейчас любят локальные инструменты.
4. **Shell command generation + подтверждение** — очень практичная фича.
5. **Многострочный ввод** + удобный `/edit` последнего сообщения.
